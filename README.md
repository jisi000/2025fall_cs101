# 2025fall_cs101
cheat sheet和学习总结

零基础开始学习，最终AC5，回顾下来觉得虽然还算可以，但是在学习方法上仍然有需要改进的地方，下学期还要继续选闫老师的数算课。 

学期初，抱着要学到真东西的心情选了闫老师的课。一开始虽然掌握不到要领，但是学习劲头很足。在学语法的时期刷了很多“超前”的题，对于很多算法一知半解，经常几小时才弄明白一道题。那时其他科目相对轻松，因此我很少看题解，大部分时间在自己研究，但是因为没有系统学习算法，学习能力也不算强，对于做过的题只能理解其本身的逻辑，无法总结出规律，进步速度比较慢。 	 

后来需要掌握的算法变得多了起来，其他科目也占据了很多时间。发现自己研究的学习效率实在很低，有做不完每日选做题和作业的风险。于是依赖题解的程度在加深，往往想不出来就直接去看题解，在开始准备cheat sheet之后，很多时候对于算法每一行代码的意义往往不去深究，而是直接背模板。这样一来，从表面上看做新题的速度的确快了起来，很多题直接套用模板很快就能AC，这样的学习习惯一直延续到了期末，我一直在刷题，这学期一共做了300题左右，但大部分题的解决方式都是在重复固定的模板套路和背新的模板，是比较机械而盲目的刷题。这样的确快，但很快暴露出了问题，由于我对于算法没有真正的自己的理解，也没有花时间总结过算法的本质逻辑，而是只看重如何应用，导致我对于“变式”的耐受度极低，需要对于模板灵活运用的题目我总是反复WA，然后看题解改代码。 	 

最终到了期末我也没有时间对我的不足进行根本的改进，依旧在企图运用刷题解决问题，最终果然没有反应过来最后一题只是加了维度的“土豪购物”，用dfs做以反复WA惨淡收场。于是我思考导致我无法深入理解算法本质，没有真正“学会”的原因，其实大部分要归咎于我一直在利用碎片时间学习写程序，除了每周的主课和上机课之外没有完整的时间可以分给计概，而主课我往往在赶作业，上机课在做模考。但我想这个问题在寒假是很好解决的，我计划在寒假学习数算的同时好好的回顾一下计概学过的东西，希望能够将这学期留的“坑”都填上，达到对算法的融会贯通。
